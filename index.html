BewitcheryUK Seller Tools is a private, self-hosted listing management application built for a small UK-based seller of vintage and collectible items. It streamlines the process of creating, scheduling, and publishing marketplace listings across eBay and Etsy, with optional cross-posting to Instagram and Facebook.

Instead of filling in marketplace forms manually, the seller describes items in plain language via a web chat or Telegram. The application drafts listings in the seller's established writing voice, and the seller reviews, edits, and explicitly approves every listing before it is published. No listing is ever created without human approval.

How It Works

1

Upload Photos

The seller sends product photos through the web dashboard or Telegram. Photos are stored on the self-hosted server and associated with the listing session.

2

Describe the Item

The seller provides a brief natural-language description ‚Äî condition, size, provenance, any special characteristics. No forms to fill in.

3

AI Drafts the Listing

The application generates a full title and description using OpenAI, written in the seller's established voice. Price and shipping are calculated based on item dimensions and seller preferences.

4

Review & Approve

The seller reads the draft and can request revisions. Only when the seller explicitly approves is the listing prepared for publication. Nothing is ever auto-published.

5

Publish & Schedule

The approved listing is published immediately or scheduled for a future date. The marketplace API (eBay or Etsy) is called to create the live listing with photos, description, price, and shipping.

Etsy API Usage

The application uses the Etsy Open API v3 for the following operations. All API calls are initiated by the seller through explicit actions ‚Äî no automated or background calls are made without the seller's knowledge.

Endpoint Purpose When Called
POST /shops/{id}/listings Create a new draft listing with title, description, price, and taxonomy When the seller approves a listing
POST /listings/{id}/images Upload product photos to the listing After listing creation
PUT /listings/{id} Update listing state from draft to active, or edit details When the seller confirms go-live
GET /shops/{id}/listings Retrieve existing listings for the seller's shop When viewing the dashboard
GET /shops/{id}/shipping-profiles Fetch available shipping profiles for new listings During listing creation
GET /seller-taxonomy/nodes Retrieve Etsy category taxonomy for categorisation During listing creation
DELETE /listings/{id} Remove a listing if the seller cancels before publication Seller-initiated deletion only

Privacy & Compliance

Data Handling

  • All data stored locally on the seller's machine
  • No buyer or customer data is collected or processed
  • No data shared with third parties beyond marketplace APIs
  • API tokens stored in local encrypted settings file
  • No analytics, tracking, or telemetry collected

API Compliance

  • Only OAuth 2.0 authenticated endpoints used
  • All API calls are seller-initiated, never automated scraping
  • Rate limits respected with built-in throttling
  • API credentials never exposed in client-side code
  • Application does not access other sellers' data

Etsy Terms

  • Listings created through official API endpoints only
  • Seller manually reviews and approves every listing
  • No automated purchasing, bidding, or account actions
  • No scraping of Etsy marketplace data or other shops
  • Seller productivity tool, not a marketplace platform

Content Policy

  • All items comply with Etsy's Prohibited Items Policy
  • Listings are for physical, tangible collectible items
  • No misleading claims ‚Äî items described honestly
  • Established, verified seller with positive feedback
  • AI descriptions reviewed for accuracy before publication
